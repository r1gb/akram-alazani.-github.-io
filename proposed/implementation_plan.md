# شرح التعديلات المقترحة وخطة التنفيذ (مختصر)

هذه الملفات تم إنشاؤها كاقتراح أولي (قبل استلام قاعدة البيانات الفعلية). تحتوي على:

1) SQL لإنشاء جداول أمان وسجلات (Users, Roles, LoginLog, AuditLog, Backups, StockAlerts, ProductBarcodes, Customer/Supplier Ledgers).
2) موديول VBA يحتوي على: حساب SHA256، دالة AuthenticateUser، ودالة AutoBackupDatabase.
3) استعلامات جاهزة للـ Dashboard (مبيعات يومية/شهرية، أرباح، عملاء نشطون، منتجات منخفضة المخزون).

كيف تستخدم هذه الملفات الآن:
- راجع أسماء الجداول والحقول في قاعدتك الحالية وكيّف SQL لتطابق البُنى الحالية (ProductID, SaleDate, TotalAmount...).
- أنصح بعمل نسخة احتياطية من القاعدة قبل تنفيذ أي CREATE TABLE.
- للأمان: نفّذ عملية Split للقاعدة (Backend + Frontend) ثم ضع Backend في مكان شبكي مؤمن.
- بعد الإضافة، استورد موديول VBA إلى الـ Frontend (Import .bas) ثم اختبر AuthenticateUser وAutoBackupDatabase على نسخة تجريبية.

خطوات مقترحة للتنفيذ العملي (عند استلام القاعدة):
1. استخراج كافة الأكواد والكائنات الحالية وتحديد التداخلات والتكرار.
2. توليد SQL محدث بأسماء الحقول الحالية ثم تنفيذ الجداول الجديدة.
3. استبدال تخزين كلمات المرور بعملية Salt+SHA256 (هنا يوجد موديل جاهز).
4. تنفيذ نظام Roles/Permissions وربطه بالـ Forms (التحقق قبل فتح Form أو إجراء عملية).
5. إنشاء Forms لعرض Dashboard واستدعاء الاستعلامات الموجودة.
6. تنفيذ نهج إشعارات المخزون (Triggers عبر الوظائف بعد البيع أو جداول وسيطة).
7. إعداد مهمة مجدولة (Task Scheduler) لتشغيل backup أو استخدام AutoBackupDatabase من Frontend عند الإقفال.
8. إعداد ACCDE للـ Frontend قبل التوزيع.

ما الذي سأفعله بعد ذلك (بمجرد أن ترفع الـ .accdb أو ملفات التصدير):
- أستخرج الأكواد الحالية وأعطي تقرير تفصيلي بالمشاكل (أخطاء، أمان، أداء، تكرار).
- أعدل SQL/VBA ليتوافق مع بنية قاعدة البيانات الفعلية وأنفذ التعديلات على نسخة تجريبية.
- أزوّدك بالأكواد المعدّلة، ملفات SQL النهائية، ودليل نشر.

أخبرني إذا أردت أن أُنشئ الآن ملف ACCDB تجريبي مُعَدّ مسبقًا أو تفضّل أن أبدأ بمراجعة قاعدة فعلية عندما ترفعها.
